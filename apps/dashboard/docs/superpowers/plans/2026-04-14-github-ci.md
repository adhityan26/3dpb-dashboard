# GitHub CI Pipeline Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Connect the repo to GitHub and set up a single Actions workflow that runs lint + test in parallel, then builds and pushes the Docker image to GHCR on every push to `master`.

**Architecture:** One workflow file (`.github/workflows/ci.yml`) with three jobs — `lint` and `test` run concurrently, `build-push` gates on both passing. Docker layers are cached via GitHub Actions Cache (`type=gha`) so warm builds take ~30s instead of ~3 min.

**Tech Stack:** GitHub Actions, Docker Buildx, GHCR (`ghcr.io`), `docker/login-action@v3`, `docker/setup-buildx-action@v3`, `docker/metadata-action@v5`, `docker/build-push-action@v6`

### Task 2: Create GitHub Actions workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the `.github/workflows/` directory**

  ```bash
  mkdir -p .github/workflows
  ```

- [ ] **Step 2: Write `.github/workflows/ci.yml`**

  Create the file with this exact content:

  ```yaml
  name: CI

  on:
    push:
      branches: [master]

  jobs:
    lint:
      name: Lint
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4

        - uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: 'npm'

        - run: npm ci

        - run: npm run lint

    test:
      name: Test
      runs-on: ubuntu-latest
      steps:
        - uses: actions/checkout@v4

        - uses: actions/setup-node@v4
          with:
            node-version: 22
            cache: 'npm'

        - run: npm ci

        - run: npx prisma generate

        - run: npm test

    build-push:
      name: Build & Push to GHCR
      runs-on: ubuntu-latest
      needs: [lint, test]
      permissions:
        contents: read
        packages: write

      steps:
        - uses: actions/checkout@v4

        - name: Log in to GHCR
          uses: docker/login-action@v3
          with:
            registry: ghcr.io
            username: ${{ github.actor }}
            password: ${{ secrets.GITHUB_TOKEN }}

        - name: Set up Docker Buildx
          uses: docker/setup-buildx-action@v3

        - name: Docker metadata
          id: meta
          uses: docker/metadata-action@v5
          with:
            images: ghcr.io/adhityan26/3dpb-dashboard
            tags: |
              type=raw,value=latest
              type=sha,prefix=sha-,format=short

        - name: Build and push
          uses: docker/build-push-action@v6
          with:
            context: .
            push: true
            tags: ${{ steps.meta.outputs.tags }}
            labels: ${{ steps.meta.outputs.labels }}
            cache-from: type=gha
            cache-to: type=gha,mode=max
  ```

- [ ] **Step 3: Commit and push the workflow**

  ```bash
  git add .github/workflows/ci.yml
  git commit -m "ci: add GitHub Actions workflow — lint + test + build-push GHCR"
  git push
  ```

### Task 3: Verify pipeline on GitHub

**Files:**
- No file changes — observation only.

- [ ] **Step 1: Open the Actions tab**

  Go to: `https://github.com/adhityan26/3dpb-dashboard/actions`

  A new workflow run should appear within seconds of the push.

- [ ] **Step 2: Confirm job graph**

  In the run detail, verify:
  - `Lint` and `Test` jobs start simultaneously
  - `Build & Push to GHCR` shows as "waiting" until both complete
  - After both pass, `Build & Push to GHCR` starts

- [ ] **Step 3: Confirm GHCR image published**

  After `Build & Push to GHCR` completes, go to:
  `https://github.com/adhityan26?tab=packages`

  Verify a package named `3dpb-dashboard` exists with two tags:
  - `latest`
  - `sha-<7-char-commit-sha>` (e.g. `sha-dc54736`)

- [ ] **Step 4: (If build-push fails with 403) Set package visibility**

  On first push, GHCR may create the package as private and require linking to the repo.
  Go to: `https://github.com/users/adhityan26/packages/container/3dpb-dashboard/settings`
  → Under "Danger Zone" → connect to repo `3dpb-dashboard`
  Then re-run the failed workflow.
